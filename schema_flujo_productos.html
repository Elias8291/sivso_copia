La pregunta clave: ¿De dónde salen los productos cada año?

El proceso tiene dos fuentes posibles al inicio de cada ejercicio:

  • Año 1 (o primera vez): El delegado selecciona productos del catálogo (propuesta) manualmente para cada empleado
  • Años siguientes: Se clonan las asignaciones del año anterior como punto de partida, y el delegado ajusta

En ambos casos, después viene el muestreo donde pueden sustituirse productos por razones de presupuesto, disponibilidad, o licitación.

El flujo completo paso a paso
1

Apertura

Se abre el ejercicio 2026. Se cargan catálogos, presupuestos y cupos por UR.

2

Asignación Inicial

Delegado clona el año anterior O selecciona de catálogo. Estado: borrador

3

Pre-muestreo

Delegado confirma las asignaciones. El sistema valida cupos. Estado: pre_muestreo

4

Muestreo

Se revisan precios reales, disponibilidad. Pueden sustituirse productos. Estado: en_muestreo

5

Aprobado

Lista final confirmada. Se genera el concentrado oficial. Estado: aprobado

→ Se hace la licitación / cotización real con proveedores → Algunos productos no están disponibles o cambiaron de precio → Se necesita sustituir: "La camisa azul ya no hay, se da la verde" → Estado cambia a en_muestreo Cuando se sustituye un producto: → producto_original_id = el producto_id actual (lo que eligió el delegado) → producto_id = el nuevo producto sustituto → es_sustitucion = true → Se recalcula precio_unitario, importe, iva, importe_total → Se revalida presupuesto con los nuevos precios La sustitución puede ser masiva: "en TODA la UR 410, reemplazar producto X por Y" O individual: solo para ciertos empleados. ══ PASO 5: APROBACIÓN ══ Actor: Admin / Jefe de dependencia Acciones: → Se revisa el concentrado final → Se aprueba → estado cambia a aprobado → Ya no se puede editar (solo admin puede revertir) → Se genera el reporte / concentrado oficial para compras
Respuesta a tu pregunta: El delegado NO selecciona "los productos de antes de la muestra" como tal. El flujo es: (1) se clonan las asignaciones del año pasado (o se hacen nuevas), (2) el delegado revisa y ajusta, (3) eso ES el pre-muestreo, (4) luego el muestreo puede sustituir algunos productos. El producto_original_id solo se llena cuando hay sustitución en el paso 4.
Estados de una solicitud_vestuario

borrador

Delegado puede editar todo: producto, talla, cantidad. No valida presupuesto aún.

↓

pre_muestreo

Confirmado por delegado. Presupuesto validado. Ya no editable por delegado (solo admin).

↓

en_muestreo

Admin puede sustituir productos. Se registra producto_original. Se recalculan precios.

↓

aprobado

Lista final. Genera concentrado oficial. Solo admin puede revertir.

¿Puede ir hacia atrás? Sí, pero controlado: un admin puede regresar de aprobado → en_muestreo, o de pre_muestreo → borrador si hay correcciones. Pero el delegado normal no puede.

Modelo actualizado
class SolicitudVestuario extends Model
{
    protected $fillable = [
        ...existentes...,
        'estado',
    ];

    // Scopes útiles para filtrar por estado
    public function scopeBorrador($q)     { return $q->where('estado', 'borrador'); }
    public function scopePreMuestreo($q)  { return $q->where('estado', 'pre_muestreo'); }
    public function scopeEnMuestreo($q)   { return $q->where('estado', 'en_muestreo'); }
    public function scopeAprobado($q)      { return $q->where('estado', 'aprobado'); }

    // Validar transiciones de estado
    public function puedeTransicionarA(string $nuevoEstado): bool
    {
        $transiciones = [
            'borrador'     => ['pre_muestreo'],
            'pre_muestreo' => ['en_muestreo', 'borrador'],
            'en_muestreo'  => ['aprobado', 'pre_muestreo'],
            'aprobado'     => ['en_muestreo'],
        ];

        return in_array($nuevoEstado, $transiciones[$this->estado] ?? []);
    }

    // Sustituir producto (solo en estado en_muestreo)
    public function sustituirProducto(Producto $nuevoProducto): void
    {
        if ($this->estado !== 'en_muestreo') {
            throw new \LogicException('Solo se puede sustituir en estado en_muestreo');
        }

        $this->producto_original_id = $this->producto_original_id ?? $this->producto_id;
        $this->producto_id = $nuevoProducto->id;
        $this->es_sustitucion = true;

        // Recalcular precios con el nuevo producto
        $precio = $nuevoProducto->precios()
            ->where('anio', $this->anio)->first();

        if ($precio) {
            $this->precio_unitario = $precio->precio_unitario;
            $this->importe = $precio->precio_unitario * $this->cantidad;
            $this->iva = $this->importe * 0.16;
            $this->importe_total = $this->importe + $this->iva;
        }

        $this->syncDenormFks();
        $this->save();
    }
}
Servicio para clonar ejercicio
class ClonarEjercicioService
{
    /**
     * Clona las solicitudes aprobadas del año anterior al nuevo año.
     * Retorna resumen de lo que se clonó y lo que necesita atención.
     */
    public function clonar(
        int $anioOrigen,
        int $anioDestino,
        int $delegacionId
    ): array {
        $solicitudesOrigen = SolicitudVestuario::where('anio', $anioOrigen)
            ->whereHas('empleado', fn($q) => $q->where('delegacion_id', $delegacionId))
            ->where('estado', 'aprobado')
            ->with(['empleado', 'producto'])
            ->get();

        $creadas = 0;
        $sinProducto = [];

        foreach ($solicitudesOrigen as $sol) {
            // Buscar producto equivalente en el nuevo año (mismo código)
            $productoNuevo = Producto::where('codigo', $sol->producto->codigo)
                ->whereHas('precios', fn($q) => $q->where('anio', $anioDestino))
                ->first();

            if (!$productoNuevo) {
                $sinProducto[] = [
                    'empleado'  => $sol->empleado->nombre_completo,
                    'producto'  => $sol->producto->descripcion,
                    'codigo'    => $sol->producto->codigo,
                ];
                continue;
            }

            $precio = $productoNuevo->precios
                ->where('anio', $anioDestino)->first();

            SolicitudVestuario::create([
                'empleado_id'     => $sol->empleado_id,
                'producto_id'     => $productoNuevo->id,
                'anio'            => $anioDestino,
                'talla'           => $sol->talla,
                'cantidad'        => $sol->cantidad,
                'precio_unitario' => $precio?->precio_unitario,
                'importe'         => $precio
                    ? $precio->precio_unitario * $sol->cantidad : null,
                'estado'          => 'borrador',
            ]);
            $creadas++;
        }

        return [
            'creadas'      => $creadas,
            'sin_producto' => $sinProducto,
            'total_origen' => $solicitudesOrigen->count(),
        ];
    }
}
Importante: La clonación crea todo en estado borrador. El delegado DEBE revisar y confirmar antes de pasar a pre_muestreo. Productos descontinuados se reportan para que el delegado elija reemplazos.
